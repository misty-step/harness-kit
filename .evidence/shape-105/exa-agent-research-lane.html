Exa Agent as an optional research acquisition lane

HIT-LIST ITEM 1 / EXA AGENT

Add Exa Agent to `/research` only as a bounded deep-research lane.

The work is not a new vendor skill. The useful shape is a cost-controlled Agent path inside `/research web-deep` for broad, multi-hop research where flat search is too shallow. Ordinary lookup, docs, news, extraction, and code-context searches stay on the existing fast provider chain.

CHOSEN DESIGN

Optional Exa Agent acquisition branch in `/research`.

Agent output is evidence with citations, run status, effort, and cost metadata. Lead synthesis still decides.

WHY IT WINS

It captures the new Exa capability without turning Harness Kit into a vendor catalog or slowing normal research.

STOP CONDITIONS

Stop if delivery needs deprecated `/research/v1`, cannot bound paid calls, cannot preserve citations, or would send private data to a non-ZDR provider without operator acceptance.

Premise source: sha256:ba4b4b... HIT-LIST.md

OUTCOME

`/research web-deep` can use a stronger multi-step source gathering lane when the task warrants it.

ACCEPTANCE

Mocked tests prove routing, run mapping, degraded failures, citations, cost metadata, and effort caps.

VERIFY

`bun test` in `skills/research`, direct premise hash check, and the full Harness Kit gate.

COMMUNICATE

Report selected route, provider evidence, skipped live Exa smoke if no key, critic result, and tree state.

DO NOT

Do not add a standalone `exa-agent` skill. Do not make live paid calls part of CI.

CURRENT STATE

`ExaProvider` posts to `/search` and returns flat links. `WebSearchOrchestrator` stops on the first nonempty provider. `/research` docs already prefer Exa for general retrieval.

NEW FACT

Exa Agent is async, multi-step, structured, costed, and cited. It has run lifecycle and continuation semantics that do not fit a flat search adapter.

PROOF SURFACE

Deterministic mocks prove behavior. Optional live smoke proves real API integration only when `EXA_API_KEY` and budget are present.

SCOPE

Goal: broad `web-deep` research gets Agent-quality acquisition when appropriate.

Non-goals: no standalone skill, no deprecated API, no default paid live calls.

Invariant: `/research` routes by capability; provider output is evidence, not authority.
