01What this is

Every skill is context tax until it proves it makes a strong model better than no skill at all. skill-eval is the generator that builds that proof — and is built so the proof can come back no-skill won.

  1. Name the one claimthe falsifiable thing the skill makes true that raw prompting doesn't
  2. Fix the task2–3 frozen fixtures that stress the claim differently
  3. Run A/Bsame model: arm A with the skill, arm B raw prompt
  4. Grade blindobjective checks first, then taste — judge can't see which is which
  5. Keep / adapt / cutthe verdict is the skill's right to exist, not a score

The grader has three tiers — and you are the gold standard, not the model:

Objective DETERMINISTIC

Scriptable pass/fail — sections present, oracle is a runnable command, cited paths resolve, no forbidden edits. Free, runs every edit. Fails without a judge.

Human GROUND TRUTH · YOU

For the judgment-heavy delta — is this spec actually buildable, is this design tasteful — your call is the truth. The keep/cut decision on a taste-heavy skill is signed off by you, or by a grader carrying a recent anchor from you.

Agent rubric CALIBRATED PROXY

A blind, decorrelated model judge lets us run the rubric cheaply and often — but it only approximates your taste and drifts. Trusted only while it matches your anchor. Proxy disagrees with you → the rubric is broken, not you.

One eval per skill. Shape's eval has 3 fixtures. The agent already graded fixtures 1 & 2 (its verdict is hidden below each until you've made your own call). The whole point of this page: get your anchor, and see whether the cheap agent judge actually tracks you.

Objective checks (free, every edit)

  • All 10 packet sections present + non-empty
  • Oracle contains a runnable command (not "it should work")
  • Repo anchors cite 3–10 paths that exist at the SHA
  • Alternatives ≥2 with verdicts + exactly one pick
  • Premise Source line present (sha256 or waiver)
  • HTML Plan path named (or waiver)
  • No forbidden edits (spec only)
  • CLI-design block present (fixture 1)

Rubric (1–5, the judgment delta)

Dimension5 looks like
Premise challengereframes the request, names the real outcome
Alternatives≥2 that fail differently + a killed-on-record pick

Pass condition: arm A beats arm B on aggregate rubric AND ties-or-wins every objective check, across ≥2 of 3 fixtures — and it isn't certified until your human anchor agrees. Cadence: cheap checks every edit · full A/B on contract change · re-audit on every model upgrade (a stronger bare model can erase a skill's edge — that's the signal to retire it).

06What your call does

This is the calibration step — the reason the page exists. Two outcomes, both valuable:

You agree with the agent → proxy is calibrated

The cheap model judge tracks your taste for this kind of call. We can let it run the frequent passes between your spot-checks, and trust the margin — with a fresh human anchor on each model upgrade.

You disagree (or tie) → the proxy is broken

If the leaner spec is more buildable to you, the rubric is wrong — or shape's claim is. Then we fix the grader and rewrite what /shape promises. We never overrule you with a model score. That's the most valuable result here.
