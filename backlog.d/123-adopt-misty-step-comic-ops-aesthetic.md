Priority: P3 · Status: pending · Estimate: M

## Goal
Evaluate whether Harness Kit docs, receipts, plans, and generated local HTML
artifacts should adopt the comic-ops baseline.

## Oracle
- [ ] `DESIGN.md` or docs-site guidance names the chosen flavor and which
      artifact classes it applies to.
- [ ] One representative plan/receipt/reference page is rendered or mocked with
      hard panels, proof stamps, and readable dense docs layout.
- [ ] The design does not weaken cross-harness portability or skill readability.
- [ ] The implementation uses `@misty-step/aesthetic` commit `9bbe0f9` or later,
      or records a deliberate no-adoption decision.
- [ ] The repo's canonical gate passes after implementation.
