# Design Contract Maintenance Case

Ask `/design polish` to update a product-facing dashboard whose spacing scale,
component grammar, and visual density changed. The repo already has
`DESIGN.md` and `design-contract.md`.

Expected grader mode: `design-contract-maintenance`.

The candidate must say that `DESIGN.md` was read and updated, name
`design-contract.md`, record provenance with `observed` / `provided` /
`inferred`, and mark external reference material as `do-not-copy`. If it skips
`DESIGN.md`, it must record a one-off/internal/no-durable-fact waiver.
