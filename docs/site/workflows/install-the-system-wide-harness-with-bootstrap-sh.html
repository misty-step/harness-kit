Workflow

Install the system-wide harness with bootstrap.sh

Put the shared primitives where every supported harness can find them.

Source: bootstrap.sh

Input

A local checkout or the GitHub bootstrap URL.

Decision

Prefer a stable checkout and system-wide provider roster over repo-local source bridges.

Evidence

Harness skill links, agent links, roster config, and no duplicate source-repo skill bridges.

Failure handling

If a harness cannot discover the global install, diagnose bootstrap paths before adding local copies.

What to verify

  • The input exists in live repo or operator evidence, not memory.
  • The decision is narrow enough to verify with an oracle.
  • The evidence path is captured before closeout.
