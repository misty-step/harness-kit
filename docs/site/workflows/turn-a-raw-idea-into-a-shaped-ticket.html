Workflow

Turn a raw idea into a shaped ticket

Move from ambition to a buildable slice with a real oracle.

Source: skills/shape/SKILL.md

Input

A rough request, repo context, constraints, and the nearest live source files.

Decision

Choose the smallest buildable slice that still makes the desired end state more true.

Evidence

A backlog.d item with goal, problem challenge, design, oracle, non-goals, and related work.

Failure handling

If the oracle is weak or the audience is wrong, route back to research or positioning before implementation.

What to verify

  • The input exists in live repo or operator evidence, not memory.
  • The decision is narrow enough to verify with an oracle.
  • The evidence path is captured before closeout.
