Choose the first workflow worth automating

Find the narrow repeatable pain before you build the agent.

Source: skills/shape/SKILL.md

Input

Messy process notes, stakeholder complaints, recent examples, and the current manual handoff.

Decision

Pick the workflow with repeated demand, visible friction, and a clear success oracle.

Evidence

Candidate workflow, non-goals, owner, acceptance proof, and failure/escalation conditions.

Failure handling

If no repeated pain or proof exists, keep researching instead of automating a demo-shaped workflow.

What to verify

  • The input exists in live evidence.
  • The decision has an oracle.
  • The evidence path is captured before closeout.
