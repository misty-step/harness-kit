Watch project signals after a deploy, release, local run, CI change, or repeated workflow. Every repo has a monitor path: production telemetry when present; otherwise healthchecks, logs, CI failures, flaky tests, benchmark drift, local daemon output, release regressions, or agent-session audit trails. Emit structured events. Escalate to /diagnose on trip, close clean on green. Thin watcher, not diagnostician.
